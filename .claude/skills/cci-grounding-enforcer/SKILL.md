# CCI/AVCS Grounding Enforcer

Il citation enforcement non è cosmetico: **è il meccanismo strutturale che rende impossibile l'hallucination libera**. Senza questo guardrail, il sistema perde l'unica differenziazione strategica rispetto a Palantir, Microsoft Copilot e Glean.

Il provider LLM di CCI/AVCS è **unico**: Claude Sonnet 4.6 via Anthropic API (`claude-sonnet-4-6`). Tutte le chiamate passano dal wrapper `cci_llm.LLMClient`, mai dall'SDK Anthropic direttamente — questo garantisce che il citation enforcer non possa essere bypassato.

## La regola assoluta

> Ogni frase generata da un LLM e destinata a un utente, a un report o a un log decisionale **deve** contenere almeno una citazione nella forma `[chunk_id]` dove `chunk_id` appartiene all'insieme dei chunk effettivamente recuperati e passati al modello.

Niente eccezioni. Niente "ma è ovvio". Niente "in questo caso non serve".

## Architettura del citation enforcer

```
LLM response  ─►  CitationParser  ─►  ┌─ ok ────►  emit to user
                                       │
                                       ├─ MISSING_CITATION  ──►  retry with reinforced prompt (max 3)
                                       │
                                       ├─ HALLUCINATED_CITATION  ──►  hard fail + audit log
                                       │
                                       └─ N retries exhausted  ──►  mark "not emittable" + HITL flag
```

## Implementazione di riferimento

Vive in `libs/cci-llm/src/cci_llm/citation_parser.py`.

```python
from __future__ import annotations
import re
from dataclasses import dataclass
from enum import Enum
from pydantic import BaseModel

CITATION_RX = re.compile(r"\[([a-zA-Z0-9_\-#]+)\]")
SENTENCE_SPLIT_RX = re.compile(r"(?<=[\.\!\?])\s+(?=[A-ZÀ-Ý])")

class ViolationType(str, Enum):
    MISSING_CITATION = "MISSING_CITATION"
    HALLUCINATED_CITATION = "HALLUCINATED_CITATION"

class Violation(BaseModel):
    sentence_index: int
    type: ViolationType
    detail: str
    sentence: str

class GroundingResult(BaseModel):
    ok: bool
    sentences: list[str]
    citations_used: set[str]
    violations: list[Violation]

class GroundingViolation(Exception):
    def __init__(self, violations: list[Violation]):
        self.violations = violations
        super().__init__(f"{len(violations)} grounding violations")

def enforce_grounding(
    response: str,
    allowed_chunks: set[str],
    *,
    allow_meta_sentences: bool = False,
) -> GroundingResult:
    """
    Validates every sentence in `response`.
    
    `allow_meta_sentences=True` lets through sentences that are
    pure formatting (e.g. "Ecco il riepilogo:") without citations.
    Default False = strict mode.
    """
    sentences = [s.strip() for s in SENTENCE_SPLIT_RX.split(response) if s.strip()]
    violations: list[Violation] = []
    citations_used: set[str] = set()

    for idx, sentence in enumerate(sentences):
        cited = set(CITATION_RX.findall(sentence))
        if not cited:
            if allow_meta_sentences and _is_meta(sentence):
                continue
            violations.append(Violation(
                sentence_index=idx,
                type=ViolationType.MISSING_CITATION,
                detail="sentence has no [chunk_id] reference",
                sentence=sentence,
            ))
            continue
        invalid = cited - allowed_chunks
        if invalid:
            violations.append(Violation(
                sentence_index=idx,
                type=ViolationType.HALLUCINATED_CITATION,
                detail=f"unknown chunks: {sorted(invalid)}",
                sentence=sentence,
            ))
        citations_used |= (cited & allowed_chunks)

    if violations:
        raise GroundingViolation(violations)

    return GroundingResult(
        ok=True,
        sentences=sentences,
        citations_used=citations_used,
        violations=[],
    )

def _is_meta(s: str) -> bool:
    # punctuation-only or starts with a bullet/marker
    return bool(re.match(r"^[\s\-\*•#\d\.]+$", s)) or s.endswith(":")
```

## Pattern di chiamata in un agente

In `services/agents/src/cci_agents/generator_agent.py`:

```python
from cci_llm.client import LLMClient
from cci_llm.citation_parser import enforce_grounding, GroundingViolation
from cci_llm.errors import NotEmittableError

async def generate_explanation(
    incoherences: list[Incoherence],
    chunks: list[Chunk],
    llm: LLMClient,                  # wraps Anthropic SDK, never the raw client
    *,
    max_retries: int = 3,
) -> Explanation:
    allowed = {c.id for c in chunks}
    system_prompt = render_prompt("prompts/v1/generator_system.j2", chunks=chunks)
    user_prompt = render_prompt("prompts/v1/generator_user.j2", incoherences=incoherences)

    for attempt in range(max_retries):
        response = await llm.complete(
            system=system_prompt,
            messages=[{"role": "user", "content": user_prompt}],
            # model is hardcoded inside LLMClient → claude-sonnet-4-6
            # prompt_caching is automatically enabled on system prompt
        )
        try:
            result = enforce_grounding(response.text, allowed_chunks=allowed)
            await audit.log_grounding_ok(attempt, len(result.citations_used))
            return Explanation(
                text=response.text,
                sources=result.citations_used,
                model_version=response.model,
                prompt_version="generator/v1.2",
            )
        except GroundingViolation as e:
            metrics.grounding_violations_total.labels(
                type=e.violations[0].type,
                model="claude-sonnet-4-6",
            ).inc()
            await audit.log_grounding_violation(attempt, e.violations)
            user_prompt = reinforce_prompt(user_prompt, e.violations)

    # all retries exhausted
    raise NotEmittableError("Claude output failed grounding after 3 retries")
```

## Il wrapper `LLMClient` — l'unico modo di parlare con Claude

`libs/cci-llm/src/cci_llm/client.py`:

```python
from __future__ import annotations
import os
from dataclasses import dataclass
from anthropic import AsyncAnthropic, APIError
from tenacity import retry, stop_after_attempt, wait_exponential_jitter

from cci_llm.pii_redaction import assert_no_raw_pii
from cci_llm.audit_hook import log_llm_call

_DEFAULT_MODEL = os.getenv("CCI_LLM_MODEL", "claude-sonnet-4-6")


@dataclass
class LLMResponse:
    text: str
    model: str
    tokens_in: int
    tokens_out: int
    stop_reason: str


class LLMClient:
    """The ONLY allowed interface to the Anthropic API in CCI/AVCS."""

    def __init__(self, api_key: str | None = None):
        # ANTHROPIC_API_KEY is loaded from Vault into the env at startup
        self._client = AsyncAnthropic(
            api_key=api_key or os.environ["ANTHROPIC_API_KEY"],
        )
        self._model = _DEFAULT_MODEL

    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential_jitter(initial=1, max=8),
        retry=lambda exc: isinstance(exc, APIError) and exc.status_code in (429, 500, 502, 503, 504),
    )
    async def complete(
        self,
        *,
        system: str,
        messages: list[dict],
        max_tokens: int = 4096,
        correlation_id: str | None = None,
    ) -> LLMResponse:
        # 1. PII guardrail: refuse to send payloads containing raw PII tokens
        for m in messages:
            assert_no_raw_pii(m["content"])
        assert_no_raw_pii(system)

        # 2. The actual API call (system message with prompt caching enabled)
        msg = await self._client.messages.create(
            model=self._model,
            max_tokens=max_tokens,
            system=[{
                "type": "text",
                "text": system,
                "cache_control": {"type": "ephemeral"},
            }],
            messages=messages,
        )

        response = LLMResponse(
            text=msg.content[0].text if msg.content else "",
            model=msg.model,
            tokens_in=msg.usage.input_tokens,
            tokens_out=msg.usage.output_tokens,
            stop_reason=msg.stop_reason,
        )

        # 3. Audit every call — never silently
        await log_llm_call(
            correlation_id=correlation_id,
            model=response.model,
            prompt_version=_extract_prompt_version(system),
            tokens_in=response.tokens_in,
            tokens_out=response.tokens_out,
        )

        return response

    async def complete_streamed(self, **kwargs):
        # Same guardrails, streaming variant — implementation in real codebase
        ...
```

**Nota**: `LLMClient` non si crea ad hoc. Esiste una sola istanza per processo (singleton DI), iniettata via `Depends(get_llm_client)` nei servizi FastAPI.

## Prompt template patterns

I prompt vivono in `services/agents/src/cci_agents/prompts/v1/` versionati. Ogni prompt **deve** contenere:

1. Una sezione `CONTEXT` con i chunk in formato `[chunk_id]: testo`
2. Un'istruzione esplicita: *"Each sentence MUST end with the citation of the chunk(s) used. Format: [chunk_id]. If no chunk supports the claim, do not write the sentence."*
3. Un esempio few-shot con citazioni correttamente posizionate

Esempio (`prompts/v1/generator.j2`):
```jinja
You are the Generator agent of the CCI/AVCS system.
You explain incoherences detected by the Coherence Engine.

# CONTEXT — Available evidence
{% for chunk in chunks %}
[{{ chunk.id }}]: {{ chunk.text }}
{% endfor %}

# INCOHERENCES TO EXPLAIN
{% for inc in incoherences %}
- Rule {{ inc.rule_id }} violated: {{ inc.description }}
  Evidence: {{ inc.evidence_chunks | join(", ") }}
{% endfor %}

# RULES (non-negotiable)
1. Every sentence MUST cite at least one chunk: [chunk_id]
2. Never invent facts not present in the chunks.
3. If you cannot support a claim with a chunk, omit the claim.

# OUTPUT
Write an explanation in Italian for the CFO.
```

## Cosa NON fare mai

| Anti-pattern | Perché è grave |
|---|---|
| `enforce_grounding(..., strict=False)` ovunque tranne in test | Disabilitare il guardrail per "fare presto" rompe la garanzia di sistema |
| Usare Claude stesso per giudicare se la citazione è valida | Self-validation by LLM è un loop di confidenza fasulla. Usa il parser deterministico. |
| Lasciare passare frasi senza citazione "perché sono di contesto" | Se è contesto, è meta — usa `allow_meta_sentences=True` esplicitamente |
| `from anthropic import AsyncAnthropic` direttamente in un agente o servizio | Bypassi il citation enforcer, le metriche, l'audit, il PII guardrail. Usa `cci_llm.LLMClient` |
| `import openai`, `import litellm`, `import ollama` ovunque | Provider diversi non sono ammessi. Solo Anthropic via `cci_llm.LLMClient` |
| Modello hardcoded come stringa nei prompt o nel codice | Deve venire da `CCI_LLM_MODEL` env var, default `claude-sonnet-4-6` |
| Logging dei prompt in chiaro senza versionamento | Impedisce reproducibilità e mappatura art. 13 AI Act |
| Inviare contenuto con PII non pseudonimizzato | Violazione GDPR. Il wrapper `LLMClient` rifiuta payload con pattern PII raw |
| Disabilitare il retry o impostarlo a 0 | Errori 429/5xx Anthropic sono transienti; un singolo tentativo dà falsi negativi |
| Stream consumer che non aggrega la risposta prima del citation check | Il citation enforcer richiede la risposta completa per validare le frasi |

## Metriche obbligatorie

Esposte da ogni servizio che chiama LLM:

```python
from prometheus_client import Counter, Histogram

grounding_violations_total = Counter(
    "cci_grounding_violations_total",
    "Claude output failed grounding check",
    ["service", "agent", "violation_type", "model"],
)
grounding_retries_total = Counter(
    "cci_grounding_retries_total",
    "Retry attempts due to grounding violations",
    ["service", "agent", "model"],
)
not_emittable_total = Counter(
    "cci_not_emittable_total",
    "Outputs marked not-emittable (all retries exhausted)",
    ["service", "agent", "model"],
)
llm_tokens_total = Counter(
    "cci_llm_tokens_total",
    "Anthropic API tokens consumed",
    ["service", "model", "direction"],  # direction: in|out
)
llm_latency_seconds = Histogram(
    "cci_llm_latency_seconds",
    "Anthropic API call latency",
    ["service", "model"],
)
```

Target operativo: `grounding_violations_total / total_responses < 1%`.
Target costo: budget giornaliero per dominio configurato in env var, alertabile a 80% di consumo.

## Test richiesti

Per ogni componente che genera output LLM, esistere DEVE:
- Un test che `enforce_grounding` rifiuta un output con citazione inventata
- Un test che `enforce_grounding` rifiuta un output senza alcuna citazione
- Un test che il retry loop si ferma dopo N tentativi
- Un test che `NotEmittableError` triggera l'audit log
