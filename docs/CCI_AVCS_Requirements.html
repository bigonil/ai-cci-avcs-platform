Sezione 1Executive Summary

CCI/AVCS è una piattaforma agentic AI di nuova generazione per la verifica continua di coerenza finanziaria, documentale e di compliance cross-dominio. Il sistema non è un chatbot RAG: è un Continuous Coherence Verification System progettato per rilevare automaticamente incoerenze tra documenti eterogenei — impegni cloud vs. budget approvati, certificazioni scadute vs. attività in corso, dati clinici vs. approvazioni etiche — senza generare affermazioni prive di evidenza.

La piattaforma è classificata come sistema AI ad alto rischio ai sensi del Regolamento UE 2024/1689 (EU AI Act), in quanto produce output utilizzati in decisioni finanziarie, compliance normativa e supervisione di sperimentazioni cliniche. Ogni componente è progettato compliance-by-design: audit log immutabile con hash chain SHA-256, human-in-the-loop obbligatorio sopra soglia, grounding forzato su ogni output LLM.

Principi Architetturali Non Negoziabili

R1 — Bounded Context Isolation

Nessun import diretto tra servizi

Comunicazione esclusivamente via REST API o eventi CloudEvents 1.0. Divieto assoluto di from services.X import da un altro servizio.

R2 — Shared Database Ban

Database separati per bounded context

Qdrant: collection separate per dominio. Neo4j: namespace separati. MongoDB: database distinti per operational e governance.

R3 — Grounding Obbligatorio

Ogni output LLM cita [chunk_id]

Il citation enforcer blocca qualsiasi completamento destinato all'utente privo di almeno un riferimento a chunk recuperato. Non disabilitabile in produzione.

R4 — Zero LLM nel Verifier

Coherence Engine 100% deterministico

Il motore di regole usa esclusivamente query Cypher temporali e valutazioni aritmetiche. Nessuna chiamata LLM per decidere se un'incoerenza esiste.

R5 — Audit Log Immutabile

MongoDB append-only + hash chain SHA-256

Collection con ruolo DB insert-only. Ogni documento include prev_hash e record_hash. Verifica integrità automatizzata via script.

R6 — Human-in-the-Loop Hard Gate

Approvazione umana obbligatoria oltre soglia

Soglia configurabile via env var (CCI_HITL_THRESHOLD_EUR). Campo motivazione obbligatorio (min 20 char). Blocco hard senza bypass.

R7 — 12-Factor Mandatorio

Configurazione via env, stateless, no segreti in repo

Credenziali esclusivamente da Vault. Stato in Redis o backing store dedicato. Graceful shutdown. Dev/prod parity tramite docker-compose.
